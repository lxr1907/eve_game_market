# EVE Project 项目总结文档

本文档总结了 EVE Project 的后端和前端技术架构及业务逻辑。

## 1. 项目概述
EVE Project 是一个为《EVE Online》（星战前夜）玩家设计的综合性工具平台，集成了市场数据分析、忠诚点（LP）收益计算、星图导航以及战斗模拟器。

---

## 2. 后端技术与业务 (Backend)

### 2.1 技术栈
- **运行时**: Node.js
- **框架**: Express 5.x
- **数据库**: MySQL (使用 `mysql2/promise` 进行原生 SQL 操作)
- **外部 API**: EVE Online ESI (EVE Swagger Interface)
- **核心依赖**:
    - `axios`: 处理外部 API 请求
    - `cors`: 处理跨域资源共享
    - `dotenv`: 环境配置管理
    - `helmet`: 安全头部加固
    - `morgan`: 日志记录

### 2.2 核心业务功能
- **数据同步 (Schedulers)**:
    - 自动从 ESI API 同步星系、星门、市场订单、忠诚点兑换项等数据。
    - 包含死锁重试机制和增量更新逻辑，确保大规模数据写入的稳定性。
- **市场分析**:
    - 提供不同数据源（晨曦 Serenity / 欧服 Tranquility / 曙光 Infinity）的市场订单查询。
    - 具备订单合并逻辑，将相同价格和位置的订单在数据库层面进行聚合。
- **星图系统**:
    - 维护星系、星座、区域及星门连接关系。
    - 提供实时星系击杀数据和在线玩家人数统计。
- **忠诚点 (LP) 收益计算**:
    - 分析忠诚点兑换项的利润，帮助玩家最大化 LP 收益。

### 2.3 项目结构
- `controllers/`: 业务逻辑控制器。
- `models/`: 数据库表结构定义及核心 SQL 操作。
- `routes/`: API 路由定义。
- `services/`: 封装外部 ESI API 调用。
- `utils/`: 包含各种数据同步脚本和数据库初始化工具。

## 3. 前端技术与业务 (Frontend)

### 3.1 技术栈
- **框架**: Vue.js 3 (Composition API)
- **构建工具**: Vite
- **UI 框架**: Element Plus
- **可视化**: 
    - `ECharts`: 用于展示星图连接、在线玩家趋势及市场波动。
    - `Phaser`: 用于实现核心页面的战斗模拟器。
- **路由**: Vue Router

### 3.2 核心业务功能
- **数据仪表盘**:
    - **市场查询 (`OrderQueryView`)**: 展示各类物品的买卖单分布。
    - **LP 利润 (`LoyaltyOfferView`)**: 列表展示物品兑换利润率。
    - **实时统计 (`OnlinePlayerStatsView`)**: 动态图表展示在线玩家变化。
- **可视化星图 (`StarMapView`)**:
    - 基于 ECharts 实现的交互式星门连接图。
    - 支持按安全等级（高安/低安/00）过滤和星系搜索。
- **战斗模拟器 (`HomeView`)**:
    - 基于 Phaser 引擎的 2D 战斗模拟。
    - 玩家可选择舰船（如阿特龙级、促进级）、装备武器（中子炮/磁轨炮）及各种功能性模块（扰频器、修炮等）。
    - 模拟接近、环绕等机动方式，实时计算速度、角速度及伤害。

### 3.3 项目结构
- `src/views/`: 各功能页面的 Vue 组件。
- `src/game/`: 存放模拟器相关的静态数据（舰船、模块、关卡配置）。
- `src/services/`: 封装 Axios 请求，与后端 API 通信。
- `src/router/`: 页面导航配置。

---

## 4. 业务价值
- **决策辅助**: 通过数据可视化，帮助商人玩家发现套利机会。
- **战术模拟**: 提供低成本的战斗练习环境，模拟不同装备配置下的战斗表现。
- **情报整合**: 整合多维度数据（击杀、人数、星图），为联盟行动提供参考。
