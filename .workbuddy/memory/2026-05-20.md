## LP收益数据重复问题修复（21:26）

- **问题**：`/api/loyalty/profit-data` 返回数据含有重复的 type_id
- **根因**：`loyalty_type_lp_isk` 表实际没有 UNIQUE KEY
  - 代码中 `createTable()` 声明了 `UNIQUE KEY unique_type_corp_region_datasource`，但 `CREATE TABLE IF NOT EXISTS` 对已存在的表不会追加索引
  - 调度器每次运行插入数据时没有约束防止重复
- **修复**：
  1. 删除全部 193 条重复行
  2. 执行 `ALTER TABLE ADD UNIQUE KEY` 手动添加唯一索引
  3. 清理 `getProfitDataWithTypeNames` 中的调试日志
- **验证**：各数据源重复组数均为 0，API 返回数据不再有重复
